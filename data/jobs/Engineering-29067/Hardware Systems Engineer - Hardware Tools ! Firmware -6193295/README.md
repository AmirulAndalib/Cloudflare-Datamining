<h3><strong>Available Locations:&nbsp;</strong>Remote - US</h3>
<h3><strong>About the Team</strong></h3>
<p><span style="font-weight: 400;">Cloudflare’s Hardware Team is hiring a Systems Engineer, with a firmware focus, in the U.S. to help deliver the Hardware team’s security goals and projects in 2024 &amp; beyond. &nbsp;</span></p>
<p><span style="font-size: 12pt;"><strong>Main responsibilities</strong></span></p>
<ul>
	<li>Maintain and improve OpenBMC firmware</li>
	<li>Design and develop features based on various industry standards such as OCP and DMTF Standards</li>
	<li>Be able to engage with vendors &amp; guide them through our requirements</li>
	<li>Design solutions for observability and configurations appropriate for various server subsystems e.g. DIMMs, NICs etc.</li>
	<li>Be able to partner with teams internally to understand HW security reqs and build a set of features / roadmap for Cloudflare</li>
	<li>Able to contribute in building/improving the HW provisioning in HW Lab … especially the steps related to BIOS, FW, settings, and HW tools.</li>
	<li>Developing and reviewing requirements/code/tests/design documents</li>
</ul>
<p><span style="font-size: 12pt;"><strong>Desired skills</strong></span></p>
<ul>
	<li>They have experience developing firmware and/or embedded SW</li>
	<li>Knowledgeable about servers, enterprise systems and hardware interfaces</li>
	<li>Familiarity with Redfish standards</li>
	<li>They are curious and very eager to learn</li>
	<li>They have a good understanding of hardware security for servers</li>
	<li>The have great communication, collaboration and very detail oriented</li>
	<li>They’re hands-on!! &nbsp; They can build firmwares from open source codes, build demos &amp; POCs, setup systems &amp; run synthetic tests, and debug complex firmware problems as needed.</li>
	<li>Experience with Yocto or Buildroot is desirable.</li>
	<li>They can create python/bash scripts and edit firmware/driver source code.</li>
</ul>
<p><span style="font-size: 12pt;"><strong>What about non-technical skills?</strong></span></p>
<ul>
	<li>They own some of their team’s Future FW or HW security tasks / goals and are responsible for delivering these goals on time with a high standard of quality</li>
	<li>They build simple, well-defined plans that are easily adapted without accruing additional complexity over time.</li>
	<li>They are expected to engage internally as domain experts in server hardware; collaborate with other teams on defining new server requirements, explain new technologies thru Wiki’s and spec sheets, and be the liaison between vendor &amp; internal customers</li>
</ul>
