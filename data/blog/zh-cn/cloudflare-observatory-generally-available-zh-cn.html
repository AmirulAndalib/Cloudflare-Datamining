推出 Observatory

2019 年，我们推出了新的 Speed Tab，其使命是解决两个关键问题：“迁移到 Cloudflare 后我的网站速度有多快？”以及“它本来可以有多快？”此选项卡允许客户比较启用 Cloudflare 功能前后的网站性能。但是，它要求用户深入分析并分析流量模式和缓存命中率以优化他们的站点，这对新的 Cloudflare 用户来说颇具挑战。

为了解决这个问题，我们开发了 Observatory，这是 Cloudflare 进行性能监控的一种新方法。Observatory 填补了以前在了解网站性能方面存在的空白，并通过提供量身定制的建议简化了解决性能问题的过程。

Observatory 集成了真实用户监控 (RUM) 数据，使用户能够了解其全球最终用户所体验到的网站性能。利用 RUM 数据，我们可以得出宝贵见解，了解网站的哪些区域可以优化，并展示可以解决这些问题的 Cloudflare 功能。

此外，Observatory 还结合了 Google Lighthouse，这是用于评估 Web 性能的行业标准工具。由于 Lighthouse 的多功能性以及在性能社区中的广泛采用，我们用 Lighthouse 替换了 WebPageTest。借助 Lighthouse，用户可以直接在 Cloudflare 仪表板中运行、安排和访问 Lighthouse 性能报告。

Observatory 还支持区域测试，认识到了解不同地点性能差异的重要性。通过模拟不同地区的网站性能，用户可以了解自己的网页在特定国家/地区的表现情况。这使用户能够为全球受众优化他们的网站，确保为全球各地的用户提供一致和快速的用户体验。

通过汇集 RUM 数据、Lighthouse 见解和区域测试，Observatory 成为您在 Cloudflare 仪表板中了解网站性能的统一位置。用户只需点击一个按钮，就可以全面了解其网站的性能，并实施基于这些数据的 Cloudflare 建议。

在 Cloudflare Observatory 中衡量性能

我们支持测试网站性能的两种主要方法，即综合测试和真实用户监控 (RUM) 测试。

综合测试涉及在受控环境下模拟用户交互和监控性能。这些测试可以提供有价值的基准测量，并有助于在部署更改之前识别潜在问题。

与之不同的是，RUM 测试会直接收集真实用户与网站交互的数据，捕捉他们在不同环境和网络条件下的实际体验。RUM 测试提供了对真正最终用户视角的洞察。通过结合综合测试和 RUM 测试，网站所有者可以了解性能的整体情况，了解更改和优化对模拟和真实用户体验的影响。

Cloudflare Observatory 将这两者结合到一个位置。Observatory 中与 Google Lighthouse 的集成为 Cloudflare 用户提供了一种综合测量和了解其网站性能的简单方法。Google Lighthouse 衡量几个影响用户体验和搜索引擎排名的关键性能指标。生成的报告提供从 1（最低性能）到 99（最高性能）的总体性能评分，使网站所有者可以轻松了解其网站的性能。

Observatory 提供了一个用户友好的界面，以红黄绿三种颜色显示每个 Lighthouse 指标，表明所测试指标的结果。一个关键指标是最大内容绘制 (LCP)，它衡量页面主要内容的加载性能。最佳 LCP 分数为小于 2.5 秒，这是用户满意的加载速度。通过 Observatory，网站所有者可以轻松查看他们的 LCP 分数和其他指标。这使他们能够优化网站的性能和用户体验。例如，通过检查 LCP 分数，网站所有者可以识别改进机会并做出明智的决策以提高其网站的性能。

新的更智能的建议

利用从 Lighthouse 和 RUM 测试中获得的见解，Observatory 可提供更加智能的建议。这使我们能够准确地识别问题，并提供量身定制的 Cloudflare 设置，以提高性能。例如，当您收到 Lighthouse 报告时，它会突出显示您的网站可以改进的地方。在提供的报告中，建议了一些图像优化增强功能。Cloudflare 会考虑这些反馈并提供产品建议，例如启用 Polish 或使用图像调整大小。这样，我们的客户只需点击一下即可提高他们的性能得分。

客户将可以在直接链接到审计的 Cloudflare 仪表板中查看这些建议，十分方便。仪表板将包含各种 Cloudflare 功能，并随着时间的推移不断改进。通过添加针对未缓存静态内容的 Cache Rules 建议和综合测试套件，用户将能够在启用特定 Cloudflare 功能之前，了解实施该功能可获得的好处。

通过在启用产品或功能之前了解其性能影响，客户可以做出明智的决策并满怀信心地优化其网站的性能。

纳入 RUM

Cloudflare 的 RUM 服务提供对用户浏览器或设备的见解，跟踪页面加载时间、响应时间和其他用户交互等指标。Cloudflare 通过其 Browser Insights 功能收集 RUM 数据，该功能将 JavaScript“信标”插入 HTML 页面。此信标从真实用户的角度向 Cloudflare 发送有关网站性能的信息，包括页面加载时间、首字节时间和其他 Web Vitals 等指标。

尽管您随时都能尝试在自己的笔记本电脑上加载几个页面并查看结果，但要考量实际的设备性能和网络条件，唯一的途径是从真实用户那里收集数据。

Observatory 现在纳入了 RUM 数据以匹配您的测试路径。这让您能够轻松了解全球范围内用户对网站的真实体验。同时，还将针对测试的路径，在 Observatory 选项卡中剖析并储存此数据。这让您能够查看综合测试数据与真实用户指标的直接对比。

我们的 RUM 提供程序已经包含 Interactive Next Paint (INP) 分数。2022 年，Google 宣布了 Interaction to Next Paint (INP) 这一新指标，将 INP 提升为新的 Core Web Vital 响应指标，取代了首次输入延迟 (FID)。FID 衡量用户与网页的首次交互与浏览器对该交互的响应之间的延迟。INP 衡量网站上任何用户交互的延迟，而不仅限于第一次输入。这一变化表明其采用一种更全面的方法来评估网站的响应能力。

如果您的 Cloudflare 区域没有启用 Web Analytics，那么我们将无法在 Observatory 中收集和显示 RUM 数据。启用此功能非常简单，可以在此处找到说明。

Observatory 现在包含一个增强的优化布局，它引入了一个一键式建议中心。在您的 Cloudflare 区域启用这些功能可增强对最新 HTTP 协议（包括 HTTP/3）的优化。此外，通过将 PNG 和 JPEG 转换为高效的 WebP 格式，改进了 Image Delivery。最后，还启用了 Cloudflare 性能工具，允许用户无缝实施 Early Hints 等新技术。这些功能旨在协助提高网站速度和整体性能。
