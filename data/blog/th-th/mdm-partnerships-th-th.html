1 นาทีที่อ่าน

ที่ Cloudflare เราคิดอยู่เสมอว่าจะหาวิธีที่จะทำให้อินเทอร์เน็ตมีความปลอดภัยขึ้น น่าเชื่อถือขึ้น และมีประสิทธิภาพมากขึ้นสำหรับลูกค้าและธุรกิจทุกขนาด การเชื่อมต่ออุปกรณ์กับแอปพลิเคชันอย่างปลอดภัยเป็นเรื่องสำคัญในด้านความปลอดภัยของแอปพลเคชันองค์กรและเพื่อความสบายใจของ CIO

เมื่อเดือนมกราคมที่ผ่านมา เรา เปิดตัวแพลตฟอร์ม Zero Trust Cloudflare for Teams ที่ปกป้องผู้ใช้ อุปกรณ์ และข้อมูลของผู้ใช้ โดยแทนที่ขอบเขตความปลอดภัยแบบดั้งเดิมด้วยเครือข่ายขอยนอกระดับโลกของ Cloudflare Cloudflare for Teams ทำให้โซลูชันการรักษาความปลอดภัย อย่างการเข้าถึงเครือข่าย Zero Trust และ Secure Web Gateway สามารถเข้าถึงได้ง่ายขึ้น สำหรับทุกๆ บริษัทไม่ว่าจะมีขนาดเท่าใด หรือแหล่งข้อมูล ซึ่งหมายความว่าการสร้างผลิตภัณฑ์ที่เป็นมิตรกับผู้ใช้มากขึ้น ใช้งานง่ายขึ้น และจัดการได้ง่ายขึ้น

Agent ของ Cloudflare WARP เข้ารหัสการรับส่งข้อมูลจากอุปกรณ์ไปที่เครือข่าย Cloudflare และลูกค้าหลายรายใช้มันเป็นองค์ประกอบสำคัญเพื่อขยายการควบคุมการปฏิเสธเริ่มต้นเป็นที่ที่ผู้ใช้งานอาศัยอยู่ วันนี้ Cloudflare ได้เปิดใช้งานเอกสารมากมายเกี่ยวกับวิธีในการใช้งาน WARP กับพันธมิตรนี้ ดังนั้น ผู้ดูแลระบบจึงมีขั้นตอนในการลงทะเบียนอุปกรณ์ทั้งหมดของคุณที่มีประสิทธิภาพและทำตามได้ง่าย

และเรายังตื่นเต้นที่จะประกาศการร่วมมือครั้งใหม่กับผู้ให้บริการการจัดการอุปกรณ์มือถือ Microsoft Intune, Ivanti, JumpCloud, Kandji และ Hexnode เพื่อให้การใช้งานและการติดตั้ง Cloudflare WARP ง่ายดายยิ่งขึ้น

MDM คืออะไร

การจัดการอุปกรณ์โทรศัพท์มือถือ (MDM) หรือบางครั้งเรียกว่าเครื่องมือการจัดการปลายทางแบบรวมศูนย์ (UEM) เสนอโซลูชันพื้นฐานในการแก้ปัญหาที่มีความท้าทายขึ้นเรื่อยๆ ในยุคแห่งการทำงานแบบกระจาย — การจัดการอุปกรณ์ทุกชิ้นขององค์กรจากแพลตฟอร์มเดียว

ยกตัวอย่างเรื่องสมมติของบริษัทที่ปรึกษาด้านการดูแลสุขภาพ เมื่อตอนตั้งบริษัท CEO ได้จ้างพนักงาานจำนวนมากจากรัฐโคโลราโดที่เธออาศัยอยู่และอนุญาตให้พนักงานใช้โทรศัพท์และแล็ปท็อปของตัวเองเพื่อเข้าถึงอีเมลและข้อมูลอื่นๆ นโยบายนำอุปกรณ์ของตัวเองมาใช้ (BYOD) อำนวยความสะดวกให้บริษัทเริ่มต้นได้ดี

หลังจากนั้น บริษัทก็มีลูกค้าที่น่าสนใจมากขึ้นพร้อมกับโปรเจกต์ขนาดใหญ่ขึ้น และเพื่อให้บริการกับความต้องการที่เพิ่มขึ้น CEO จึงเริ่มจ้างงานทั่วทั้งสหรัฐ และใช้งานอุปกรณ์ขององค์กร ยิ่งไปกว่านั้น ลูกค้ารายนี้มีมาตรฐานที่เข้มงวดมากเกี่ยวกับการจัดการข้อมูลความลับของผู้ป่วย

ที่ปรึกษาของเราจึงรู้สึกว่าต้องเพิ่มระดับการรักษาความปลอดภัยให้สูงขึ้น แต่การใช้อุปกรณ์หลากหลายที่กระจายอยู่ทั่วประเทศ CEO จะทำอย่างไรเพื่อพัฒนาความสามารถในการมองเห็นอุปกรณ์ที่ได้รับการจัดการและไม่ได้รับการจัดการ เพื่อตรวจสอบว่ามีการอัปเดตที่เหมาะสม ไม่ตกอยู่ในความเสี่ยงหรือสูญหายไป หากอุปกรณ์ตกอยู่ในความเสี่ยงหรือเป็นอันตราย เราจะทำอย่างไรเพื่อล้างอุปกรณ์นั้นจากระยะไกล เพื่อไม่ให้ข้อมูลของลูกค้าหรือบริษัทรั่วไหล

โซลูชัน MDM สามารถตอบคำถามนี้ได้ มันถูกสร้างมาโดยเฉพาะเพื่อนโยบายการกำหนดค่าว่าผู้ใช้สามารถทำอะไรกับอุปกรณ์ได้ เปิดใช้การอัปเดตระบบดำเนินการ และติดตั้งซอฟต์แวร์ใหม่ -- ทั้งหมดเกิดขึ้นพร้อมกับที่อนุญาตให้ทีม IT มองเห็นอุปกรณ์ทั้งหมดได้เป็นหนึ่งเดียว โดยปกติปัญหานี้คงจะแก้ได้โดยการรับความช่วยเหลือจากฝ่าย IT แต่ตอนนี้เราสามารถจัดการได้จากระยะไกลในทุกระดับ

ทำให้อุปกรณ์ของลูกค้าใช้งานได้อย่างมีประสิทธิภาพ

Cloudflare รู้ว่าองค์กรอย่างที่ปรึกษาด้านการดูแลสุขขภาพด้านบนต้องการที่จะเพิ่มความปลอดภัยและการมองเห็นผู้ใช้ที่กระจัดกระจายได้ WARP อุปกรณ์ของลูกค้าเราจะเข้ามาช่วยในเรื่องนี้โดยเปิดใช้งานการยืนยันตัวตนและการบังคับใช้นโยบายอุปกรณ์แบบ posture-aware ที่ปลายทาง

เราได้เพิ่มประสิทธิภาพลูกค้าของเราเพื่อให้สามารถใช้งานวิธีการได้หลากหลาย ดังนั้นองค์กรจึงมีความยืดหยุ่นตามที่ต้องการเพื่อเปิดใช้งานความสามารถ Zero Trust ของ Cloudflare for Teams ได้อย่างง่ายดาย ตัวอย่างเช่น WARP ทำงานทั่วระบบการดำเนินการหลักทั้งหมด (เช่น Windows, MacOS, Linux, chrome OS, iOS, and Android) และหากไม่พิจารณาเรื่องการใช้งานด้านกลไล WARP ใช้ ปัจจัยที่มีร่วมกัน เพื่อให้ผู้ดูแลของคุณมีประสบการณ์ที่สอดคล้องกัน

เพื่อแสดงให้เห็นการใช้งานที่มีประสิทธิภาพ นี่เป็นการจำลองสถานการณ์ทั่วไปเกี่ยวกับวิธีการใช้งานลูกค้าของเราบน Windows โดยมีการเปลี่ยนแปลงเพียงเล็กน้อยในบรรทัดคำสั่ง

1. หากคุณต้องการใช้กฎการคัดกรอง HTTP Browser Isolation หรือทำอะไรก็ตามกับ Device Posture สิ่งสำคัญคือการยืนยันตัวตนผู้ใช้ต่อ Teams Organization และส่งข้อมูลการรับส่งข้อมูลไปที่ WARP

Cloudflare_WARP_Release-x64.msi /quiet ORGANIZATION="exampleorg" SERVICE_MODE="warp"

2. หากคุณไม่สนใจเกี่ยวกับการยืนยันตัวตนและต้องการติดตั้งแบบเงียบๆ เหมือนกับสถานการณ์ด้านบน คุณต้องใช้โทเค็นบริการ และปิดการใช้งาน UI เริ่มต้นของลูกค้า

Cloudflare_WARP_Release-x64.msi /quiet ORGANIZATION="exampleorg" SERVICE_MODE="warp” AUTH_CLIENT_ID=”” AUTH_CLIENT_SECRET=”” ONBOARDING=”false”

3. พนักงานของคุณมีการเดินทางไปในประเทศหรือสถานที่ที่ไม่อนุญาตให้มีการรับส่งข้อมูลเข้ารหัสใน tunnel หรือไม่? คุณสามารถให้พวกเขาปิด WARP แต่ก็ยังต้องปฏิบัติตามกฎ DNS ของบริษัท

Cloudflare_WARP_Release-x64.msi /quiet ORGANIZATION="exampleorg" SERVICE_MODE="warp” MODE_SWITCH=”true”
