網際網路在最近五年發生了巨大變化，這已是老生常談。分散式帳本、非同質化代幣 (NFT) 和跨平台元宇宙 (metaverses) 等新技術已風行一時。除非您經常與香港、舊金山和倫敦的 Web3 社群交流，這些技術對於一般開發人員而言門檻比較高。您必須要明白如何運行分散式節點、設置深奧的開發環境，並且緊跟最新潮流，才能讓您的應用程式順利運行。這一切都在今天畫上句點。現在，您可以報名參加封測，從乙太坊和 IPFS 閘道開始，體驗我們的 Web3 產品。

在更深入之前，需要先簡介區塊鏈 (以乙太坊為例) 和 InterPlanetary FileSystem (IPFS)。在 Web3 環境中，您可以將乙太坊視為運算層級，將 IPFS 視為儲存層。透過利用去中心化分類帳技術，乙太坊提供了可驗證的去中心化運算。使用者可以將稱為「智慧合約」的公開二進位檔案實體化，對一組不可變的記錄執行操作。這一組記錄是區塊鏈的狀態。必須由網路上的各個節點進行維護，使得它們能夠驗證並參與運算。因此，對許多資料執行操作的成本會很高。常見的模式是使用 IPFS 作為外部儲存解決方案。IPFS 是一種在分散式檔案系統上儲存內容的點對點網路。內容由其雜湊來標識，因此只需很小成本就能在區塊鏈脈絡中加以參考。

如果您想要深入瞭解 Web3 運作原理，不妨讀一下我們其他有關 Web3 定義和用 Cloudflare Workers 建立 Web3 Dapp 的部落格貼文。

Web3 和元宇宙

在過去四年中，我們一直努力發展必要的技術，以在全球規模上存取 Web3 服務，而元宇宙的概念再度流行起來。此概念因《潰雪》和《一級玩家》等作品而得以普及，它並不複雜。試想這樣一個網際網路，您可以躍進一個應用程式裡，能存取所有您喜愛的數位產品，不論何處購得，均可使用。您可以在社交媒體上出售作品，不必授予全球許可，而且買家能夠在自己的線上遊戲中使用這些作品。在元宇宙中，版權和所有權可以透過儲存在 IPFS 中的 NFT (非同質代幣) 管理，並透過乙太坊無信任存取。在這樣的世界中，不論創作者採用什麼平台，每天都能從自己的內容中獲利，並將內容提供給每個人使用，因為內容並不是儲存在圍牆花園之中，而是儲存在採用開放標準的去中心化生態系統內。

這改變了使用者和內容創作者對網際網路的思考。例如這樣的問題：「您是否需要模型-視圖-控制器系統和伺服器才能構建應用程式？」、「如何才能在不同平台之間為 Web 資源提供一致的名稱？」，以及「我們需要讓資料封鎖在其他公司的系統背後，還是讓最終使用者擁有他們的資料？」這產生了不同的信任推斷。不再因為一家公司是您的使用者資料唯一擁有者而信任他們，而是利用可由所有參與者驗證的來源以建立信任。他們可以是您透過訊息應用程式實際互動的人、登入公共憑證透明度某個網站證書日誌的 X.509 憑證，或者與分散式應用程式的區塊鏈互動的公開金鑰。

這是一個激動人心的時刻。然而，與網際網路興起時不同，現有大型知名企業想要控制 Web3 和這個元宇宙的塑造與發展方向。我們相信未來屬於去中心化私有網路，網路是基於開放標準，並獨立於任何公司或集中力量之外。我們自信能夠成為眾多技術平台之一，支援 Web3 和正在崛起的元宇宙生態系統。正因為如此，我們非常高興宣佈啟動乙太坊和 IPFS 閘道封測。這些技術是 Web3 和日漸興盛的元宇宙的前鋒。

Cloudflare Web3 閘道隆重登場！

閘道是位於用戶端 (例如您的瀏覽器或行動裝置) 和一系列其他系統之間的電腦，可以幫助將流量從一種通訊協定轉換為另一種，使得驅動應用程式所需的系統能夠正確地處理相關請求。不過，現存的閘道有許多不同類型。

您或許聽說過 API 閘道，它負責接收傳入應用程式的 API 呼叫，集合適當的服務來滿足這些請求，並將正確的反應回傳給最終使用者。每次觀看 Netflix 時您都會用到閘道！這家公司利用 API 閘道來確保存取其串流服務的數百種不同裝置可以成功收到正確的回應，以便最終使用者能夠觀看他們的節目。閘道是這個星球上每一個最終使用者走進 Web3 世界所需的一個關鍵元件。

請記住，Web3 或分散式 Web 是一套以無伺服器方式代管內容和 Web 應用程式的技術，純粹利用分散式系統和取得共識的通訊協定。透過閘道，您可以在瀏覽器中使用這些應用程式，而不必安裝外掛程式，或運作單獨的軟體 (稱為節點)。運作分散式 Web 的社群也遇到了同樣的難題：需要穩定、可靠和有韌性的方法，將 HTTP 請求轉換為正確的 Web3 函數或通訊協定。

今天我們隆重推出 Cloudflare 乙太坊和 IPFS 閘道，以幫助 Web3 開發人員發揮最大潛力來開發應用程式，不必費心去運行必要的基礎結構來支援乙太坊 (Eth) 或 IPFS 節點。

現有的乙太坊和 IPFS 閘道有什麼問題？

HTTP 等傳統 Web 技術經過了數十年發展，形成能讓網站變得快速、安全並可供使用的標準和最佳作法。它們的開發不是著眼於網際網路中的分散式 Web 這一塊，對冗餘的關注度不足。我們抓住這機會，透過建構閘道將 Web 最佳化，並將基礎結構引向分散式 Web。閘道是一種服務，將 HTTP API 呼叫轉換為 IPFS 或乙太坊函數，同時在 HTTP 層面新增 Cloudflare 增值服務。客戶可以透過 Cloudflare 使用單一控制台來操作他們的整個網路控制層級，這將產生巨大的不同。您可以從一個位置上管理 DNS、防火牆、負載平衡、限速和通道，還有行銷網站、分散式應用程式 (Dapp) 和公司網路安全。

對許多客戶而言，現有的 Web3 閘道解決方案沒有規模足夠大的網路，無力處理乙太坊和 IPFS 網路中數量日益增多的請求。但更為重要的是，它們沒有適當程度的韌性和冗餘以達到商業的期望和大規模運作的要求。分散式 Web 的概念正是為此而生：保持分散，不會有單一方來掌控整個市場。速度、網路安全和可靠性是我們的中心思想。我們很榮幸參與這個蓬勃發展的 Web3 基礎結構社群，從而幫助 Dapp 開發人員從基礎結構提供者處獲得更多的選擇、更高的擴充性和可靠性。

舉一個顯而易見的例子，當現有的閘道服務中斷時，如果處理流量的閘道太少，這種停機的結果便是預處理交易落後於要存取的區塊鏈，從而導致交易等候時間加長，並可能造成交易失敗。更糟糕的是，當去中心化應用程式 (Dapp) 開發人員使用 IPFS 驅動其前端時，它可能會導致整個應用程式發生故障。總之，這會讓企業和終端使用者都灰心喪氣，無法獲得產品或服務的收入，因此讓一部分業務出現停滯，並失去最終使用者的信任，他們需要這些服務運作可靠，以管理其 Web3 資產。

Cloudflare 如何解決這個問題？

我們發現了這個獨特的機會，Web3 社群中有一部分恰巧與 Cloudflare 的傳統客戶群對應：分散式 Web。這個部分存在一些重大的可用性問題，而 Cloudflare 藉著可靠性、效能和快取，可協助解決這些問題。Cloudflare 擁有全球網路，是這個空間內其他公司都無法企及的優勢，甚至整個行業中也鮮有人可以匹敵。例如，透過我們 IPFS 閘道提取的內容可以快取儲存在使用者附近，使下載延遲縮短到幾毫秒。與之相比，使用原生 IPFS 時每筆資產的延遲可達幾秒鐘。這一速度足以讓基於 IPFS 的服務向混合式遷移。內容可以透過去中心化通訊協定從來源提供 (用於存取它們的瀏覽器和工具也日趨成熟)，並藉由 Cloudflare 等閘道提供給普通 Web 使用者。我們提供了一種方便、快速又安全的方案來瀏覽這種分散式內容。

在乙太坊上，使用者可以劃分成兩類：運行智慧合約的應用程式開發人員，以及希望使用上述合約進行互動的使用者。智慧合約基於其代碼匿名運行，使用者則必須獲取資料並發送交易。作為鏈的一部份，智慧合約不必考慮網路或使用者介面是否上線。正因為此，去中心化交換可以跨越多個介面連續運行而不會中斷。另一方面，使用者確實需要知道鏈的狀態，並且能夠與之互動。因此，應用程式開發人員必須要求使用者運行乙太坊節點，或者透過標準化 JSON RPC API 指引他們使用遠端節點。這是 Cloudflare 的用武之地。Cloudflare 乙太坊閘道依賴乙太坊節點，並提供安全、快速的介面來連線乙太坊網路。它允許應用程式開發人員在前端應用程式中利用乙太坊。閘道可以與乙太坊鏈的任何內容部件互動，包括 NFT 合約、DeFi 交換，或 ENS 等名稱服務。
