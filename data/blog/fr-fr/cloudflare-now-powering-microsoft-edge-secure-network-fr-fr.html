Entre les cookies de tiers qui suivent votre activité sur les sites web et la publicité hautement ciblée basée sur votre adresse IP et vos données de navigation, ce n'est un secret pour personne que la navigation sur Internet n'est pas aussi privée qu'elle devrait l'être.Chez Cloudflare, nous pensons que tout le monde devrait pouvoir naviguer sur Internet à l'abri de toute forme de traçage et de surveillance.

C'est pourquoi nous sommes ravis d'annoncer que nous nous sommes associés à Microsoft Edge pour fournir un VPN rapide et sécurisé, directement dans le navigateur. Les utilisateurs n'ont pas besoin d'installer quoi que ce soit de nouveau ou de comprendre des concepts complexes pour bénéficier des dernières avancées en matière de confidentialité au niveau du réseau : le VPN Edge Secure Network est disponible dans la dernière version grand public de Microsoft Edge sur la plupart des marchés, et est automatiquement fourni avec 5 Go de données. Il suffit d'activer la fonction en allant dans [Microsoft Edge Paramètres et plus (…) > Éléments essentiels du navigateur, et de cliquer sur Obtenir un VPN gratuitement]. Pour plus d'informations, consultez la page Edge Secure Network.

La plateforme Privacy Proxy de Cloudflare n'est pas un VPN classique

Pour prendre un peu de recul : un VPN est un moyen par lequel le trafic Internet quittant votre appareil passe par un serveur intermédiaire géré par un fournisseur - dans ce cas, Cloudflare ! Il existe de nombreux éléments importants qui rendent cela possible, mais l'un d'entre eux est le protocole VPN, qui définit la manière dont le tunnel est établi et dont le trafic circule à travers lui. Vous avez peut-être entendu parler de certains de ces protocoles : Wireguard, IPsec et OpenVPN, par exemple. Et bien que nous ne soyons pas étrangers à ces protocoles (l'implémentation WireGuard de Cloudflare est actuellement utilisée par des millions d'appareils qui utilisent 1.1.1.1+WARP) – nous considérons notre plateforme Privacy Proxy comme un moyen de faire avancer la prochaine frontière de la confidentialité sur Internet et d'embrasser l'une des valeurs fondamentales de Cloudflare : les normes Internet ouvertes.

La Privacy Proxy Platform met en œuvre HTTP CONNECT, une méthode définie dans la norme HTTP qui assure la procuration du trafic en établissant un tunnel et en envoyant ensuite des flux d'octets fiables et ordonnés par l'intermédiaire de ce tunnel. Pour en savoir plus sur cette méthode de proxy (et son histoire !), consultez notre Guide d'introduction aux proxys.

Nous nous appuyons également sur d'autres éléments de l'infrastructure de Cloudflare axée sur la confidentialité qui sont déjà déployés à grande échelle : les requêtes utilisent d'abord 1.1.1.1 pour le DNS, un proxy à jeton basé sur Privacy Pass pour l'authentification du client, et Geo-egress pour choisir une adresse IP de sortie précise sans exposer l'emplacement exact des utilisateurs.

Fonctionnement

Penchons-nous sur les détails de ces composants. Pour les besoins de ce blog, nous appellerons les appareils que les gens utilisent pour naviguer sur l'internet (votre téléphone, votre tablette ou votre ordinateur) des clients, et les sites web qu'ils essaient de visiter des sites d'origine.

La plateforme Privacy Proxy comprend trois parties principales :

  1. Token Proxy : il s'agit du service qui vérifie si vous êtes un utilisateur Edge Secure Network disposant d'un compte Microsoft légitime.
  2. Privacy API : sur la base de ce qui précède, l'API de confidentialité de Cloudflare émet des jetons d'authentification que les clients utilisent pour s'authentifier auprès du proxy lui-même.
  3. Privacy Proxy : il s'agit du service proxy basé sur HTTP CONNECT fonctionnant sur le réseau de Cloudflare. Ce service vérifie que le client présente un jeton d'authentification valide et, le cas échéant, transmet la requête HTTP cryptée au site d'origine. Il est également responsable de la sélection d'une adresse IP de sortie valide à utiliser.

Lorsque les protections Edge Secure Network sont activées – par exemple, lorsqu'un utilisateur se connecte à un réseau Wi-Fi ouvert dans un café - notre proxy demande automatiquement à ce client un jeton pour s'authentifier. Si le client dispose d'un jeton, il en présentera un. Dans le cas contraire, il utilisera le proxy de jetons pour créer un nouveau pool avec l'aide d'un contrôleur et d'un émetteur : le contrôleur vérifie la validité du client et du compte Microsoft, et l'émetteur émet des jetons pour ce client en retour. Cette danse est basée sur le protocole Privacy Pass. Il est important de noter qu'il permet à Cloudflare de valider que les clients sont bien ceux qu'ils prétendent être sans collecter ou stocker les données personnelles des utilisateurs de Microsoft.

Une fois que le client a présenté au serveur proxy un jeton valide, le proxy de confidentialité choisit une adresse IP de sortie valide sur la base d'un hachage de la géolocalisation du client. Il utilise ensuite l'enregistrement DNS (fourni par le résolveur DNS de Cloudflare, 1.1.1.1) pour ouvrir une session cryptée vers le site web d'origine. À partir de là, c'est assez simple : si l'utilisateur continue à naviguer sur ce site, d'autres requêtes seront envoyées via cette connexion ; s'il arrête ou ferme son navigateur, cette connexion sera également fermée.

Parce que Cloudflare transmet des millions de requêtes par seconde, de nombreux aspects opérationnels du proxy sont gérés par Oxy, notre cadre de proxys qui s'occupe de tout, de la télémétrie, des redémarrages en douceur, du multiplexage de flux et des fallbacks IP, ainsi que des crochets d'authentification.

Faible latence sur le dernier kilomètre et parité de géolocalisation grâce au réseau de Cloudflare

La mise en œuvre du proxy de confidentialité de Cloudflare maximise l'expérience de l'utilisateur sans sacrifier la confidentialité. Lorsque Edge Secure Network est activé, les utilisateurs obtiennent des résultats de recherche et de navigation correspondant à leur situation géographique. Chez Cloudflare, nous appelons cela le test de la pizza : les gens doivent pouvoir utiliser n'importe lequel de nos produits de proxy de confidentialité et obtenir des résultats pour « pizzerias près de chez moi ». Nous y parvenons en passant toujours par un centre de données Cloudflare dont l'adresse IP correspond à la localisation de l'utilisateur – nous avons écrit davantage sur la façon dont nous avons procédé pour la version 1.1.1.1+WARP.

Contrairement à l'opérateur VPN typique qui possède des dizaines - voire des centaines - de serveurs, Cloudflare a une empreinte beaucoup plus large : des datacenters dans plus de 300 villes. Parce que notre réseau est une approche Anycast « chaque service, partout », chacun de nos datacenters peut accepter le trafic d'un client du réseau Edge Secure. Cela signifie que les utilisateurs Edge détecteront automatiquement un datacenter Cloudflare géographiquement très proche d'eux et s'y connecteront, minimisant ainsi la latence sur le dernier kilomètre. Enfin, comme Cloudflare exploite également un CDN, les sites web qui sont déjà sur Cloudflare bénéficieront d'un « hot-path » et se chargeront plus rapidement.

Chez Cloudflare, nous nous efforçons toujours d'apporter plus d'options de confidentialité à l'Internet ouvert, et nous sommes ravis d'offrir une navigation plus privée et plus sûre aux utilisateurs de Edge. Pour en savoir plus, rendez-vous sur la page Edge Secure Network de Microsoft ou sur la page d'assistance de Microsoft.
